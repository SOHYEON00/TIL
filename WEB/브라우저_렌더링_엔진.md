1. **HTML, CSS 파싱**
    - 파싱?

        브라우저가 코드를 이해하고 사용할 수 있는 구조로 변환하는 것.

    - 파싱 이후
        - html 파서 → DOM 트리
        - css 파서 → CSSOM 트리
2. **렌더 트리 구축**
3. **렌더 트리 배치**

    = 각 노드가 화면의 정확한 위치에 표시되는 것을 의미한다.

4. **렌더 트리 그리기**

- 모두 점진적으로 진행된다.

    즉, 더 나은 사용자 경험을 위해 네트워크로부터 나머지 내용이 전송되길 기다리지 않고, 받은 것부터 먼저 파싱 → 배치 → 그리기 시작한다.
